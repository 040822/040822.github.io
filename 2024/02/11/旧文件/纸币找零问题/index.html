在贪心算法中的讨论

有1元、2元、5元、10元的纸币分别有a[0], a[1], a[2], a[3]张，要用这些纸币凑出m元，至少要用多少张纸币？

联系一下生活实际，当m>10时，我们会先拿出10元的纸币，然后再拿出5元的纸币，然后...

m=int(input())
a=list(map(int,input().split()))
c=[1,2,5,10]
n=0
def num(m,a,c):
    b=0
    while m>=c:#判断总金额是否大于币值
        if b!=a:#判断该币值纸张是否已用完
            m-=c
            b+=1
        else:
            return b,m
    return b,m
for x in range(4):
    d,m=num(m,a[3-x],c[3-x])
    n=n+d

有1元、5元、7元的纸币分别有a[1], a[2], a[3]张，要用这些纸币凑出m元，至少要用多少张纸币？

似乎将上边的代码改一改就行了。

 

分析一下：上边的思想是优先使用大额纸票。那么大额优先就一定是最优解吗？

设只有1、2、5纸票，上边的结果是a、b、c，则有a+2b+5c=m

假设小额优先为最优解，则结果为x、y、z,必有x+y+z<a+b+c且z<c

由于非5元优先时，我们会把5元换成小额钞票，则有关系式：5△5（5元减少的）=2△2+1△1

易知△5<△2+△1

所以说这种情况不存在，大额优先最优解。

于是这里大额优先是一种正确的贪心选择。

然而在情况二中，有反例：m=10时，大额优先时:7+1+1+1=10,n=4

而5+5=10,n=2

于是大额优先时不正确的贪心选择。

原因分析如下：（以下均为咱自己想的）

造成这种情况的原因是因为（咱也说不清楚）

1,2,5中，m<2*5（两个5）且n最大的情况为9=5+2+2(特例),8=5+2+1;m<2*2且n最大的情况是3=2+1 ;m<2*1略

1,5,7中,m<2*7 : 11=7+1+1+1+1=5+5+1 ; m<2*5 : 9=5+1+1+1+1=7+1+1; m<2*1略。

②中存在7+1+1=5+5;5+1+1=7的最优转换，存在5，7优先达不到最优的情况；而①中最优转换为5=2+2+1;2=1+1，不存在5优先达不到最优的情况。

下边继续深入，对两种情况的临界进行分析：①1,3②4,2

猜想：当临界值递增（包含临界值相等）时大额优先最优。(咱也不会证)

定义：设当m<2*c[x](面额)，最优解的n最大时的情况成为10086情况.

易证:临界值递增时10086情况下m=c[x]-1（咱也不会证）
